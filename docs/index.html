Four tools in one

Most PKI tools do one thing. notafter does four — replacing openssl, zlint, manual OCSP checks, and nonexistent PQC tooling.

7x

Certificate Linting

Seven checks on every cert in the chain: expiry, key strength, signature algorithm, SAN validation, self-signed detection, chain completeness, and TLS version.

3x

Revocation Checking

OCSP responder queries, CRL download and serial lookup, Certificate Transparency log checks via crt.sh. All three methods, every scan.

PQ

PQC Readiness Scoring

0-10 quantum readiness score with letter grade. Tracks CNSA 2.0 compliance deadlines. Covers ML-DSA, SLH-DSA, ML-KEM, and IETF composite hybrids.

BOM

Cryptographic Inventory

CycloneDX 1.6 CBOM output. Machine-readable inventory of every algorithm, key, and certificate in a TLS connection. Fleet-wide aggregation supported.

Command reference

Two commands cover everything — single hosts and fleet-wide scans.

notafter scan

Audit a single host or certificate file.

CommandDescription
notafter scan example.comFull audit: lint + revocation + PQC scoring
notafter scan example.com:8443Non-standard port
notafter scan --file cert.pemAudit a PEM or DER certificate file
notafter scan example.com --jsonJSON output for CI/CD pipelines
notafter scan example.com --cbomCycloneDX 1.6 cryptographic bill of materials
notafter scan example.com --no-revocationSkip OCSP, CRL, and CT checks
notafter scan example.com --no-pqcSkip PQC readiness assessment

notafter fleet

Bulk scan from a host file or CIDR range with async concurrency.

CommandDescription
notafter fleet hosts.txtScan all hosts in a file (one per line)
notafter fleet 10.0.0.0/24Scan a CIDR range (up to /16)
notafter fleet hosts.txt --jsonJSON report for all hosts
notafter fleet hosts.txt --cbomFleet-wide cryptographic inventory
notafter fleet hosts.txt -c 100 -t 5100 concurrent connections, 5s timeout

Exit Codes for CI/CD

0All checks passed
1Warnings found
2Critical issues or connection errors

PQC readiness scoring

Every scan produces a quantum readiness score from 0 to 10, graded across six weighted factors.

3
Key Exchange
Hybrid PQC KEM required
2
Leaf Certificate
PQC or hybrid signature
2
Chain Certs
All intermediates + root PQC
1
TLS Version
TLS 1.3 required for PQC
1
Clean Baseline
No SHA-1, MD5, or DSA
1
CNSA 2.0
All algorithms approved
A 9-10 — Quantum-safe
B 7-8 — Hybrid protection
C 5-6 — Partial readiness
D 3-4 — Minimal
F 0-2 — Vulnerable

CNSA 2.0 deadlines

Federally mandated migration timeline for national security systems. notafter tracks compliance against each milestone.

End of 2025
Software/firmware signing must support CNSA 2.0
End of 2026
Networking equipment must support CNSA 2.0
2027
New NSS acquisitions must be CNSA 2.0 compliant
2030
Software signing + networking exclusively CNSA 2.0
2033
Web browsers/servers exclusively CNSA 2.0
2035
All National Security Systems fully quantum-resistant
